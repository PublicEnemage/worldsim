## Open CRITICAL/HIGH requiring investigation

### DEMO-168: Frame A Form 1 shows 0.85, contradicting FOUND state 0.83

**Root cause hypothesis:** The spec navigates to the ZMB scenario via URL `?scenario=<id>`. The scenario may have been created with `fiscal_multiplier: 0.85` applied in Form 1. The constraint search result (0.83) is a Form 3 output — a separate mechanism from the Form 1 policy input. The display presents both simultaneously.

**Investigation needed:** Confirm whether the spec applies a Form 1 input of 0.85 before the constraint search, or whether the Form 1 state comes from the persisted scenario configuration. If the latter, the fix is to clear or reset Form 1 before capture, or to load the scenario with `fiscal_multiplier: null` in the demo fixture.

**Demo-day risk:** HIGH — a technically literate audience sees two different numbers on the same column.

---

### DEMO-178A / DEMO-181: TERMINAL Reserve Coverage breach in Frame A

**Root cause hypothesis:** The walkthrough honest disclosures state "ExternalSectorModule is disabled for Act 1." If ExternalSectorModule is correctly disabled, Reserve Coverage (months) should not produce an MDA alert. Its presence suggests either (a) the module is not disabled as stated, or (b) Reserve Coverage is computed by the Financial module independently of ExternalSectorModule.

**Investigation needed:** Check the ZMB Act 1 scenario fixture configuration for ExternalSectorModule enable state. Check which module produces the Reserve Coverage MDA threshold.

**Demo-day risk:** HIGH — the TERMINAL BREACHED alert co-existing with the "safe boundary FOUND" state requires immediate explanation. The walkthrough has no scripted response for this combination.

---

### DEMO-200: HCL Bottom Quintile entry shows dash (no value)

**Root cause hypothesis:** The Human Cost Ledger strip's "Bottom Quintile" indicator either (a) reads from a different data path than Zone 1B CohortImpactSection, or (b) the indicator definition in the HCL strip does not match the cohort ID used in Zone 1B's focal cohort.

**Investigation needed:** Inspect the HCL strip component to identify which data field/path it uses for "Bottom Quintile", and compare to Zone 1B CohortImpactSection's focal cohort value.

**Demo-day risk:** HIGH — the founding principle requires equal visual weight for human cost indicators. The constraint-floor search protects the bottom quintile; if the HCL ledger cannot display the value it protects, the principal-level display is broken.

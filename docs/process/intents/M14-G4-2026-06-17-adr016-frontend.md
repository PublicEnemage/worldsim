## 2. Persona Trace Elements Targeted

> *Derived from ADR-016 §Persona and UX Traceability.*

**P-1 — Persona served:**
Primary: Persona 2 — Finance Ministry Negotiator (Zambian ministry analyst archetype).
The negotiator building a scenario for a sovereign debt restructuring session (Preparatory
entry state) and responding to input challenges at the table (Reactive entry state).

Secondary: Persona 3 — Policy Analyst / Country Economist (Kofi Otieno archetype).
The analyst inheriting a colleague's scenario and needing to know what parameters and
data were applied (Retrospective entry state).

**P-2 — Entry state:**
- **Preparatory** (primary): Persona 2 opens the scenario creation form to model Zambia
  2024. G4 delivers: entity selector showing ZMB, and a data quality preview that
  immediately signals "Financial — T2 · IMF BOP 2024-Q1 · Ecological — T4 (synthetic —
  SADC comparables)." She knows before committing to the scenario which frameworks are
  IMF-observed vs. synthetic inference. Time ceiling: 5 minutes including data quality
  review per ADR-016 P-2.
- **Reactive** (primary): Persona 2 is at the table. A creditor challenges her current
  account assumption. She clicks "Grounding ▼" (one interaction from Zone 0). Within
  3 seconds she reads "Reserve coverage: 7.1 months · CBJ 2023-Q4 · T2." She responds
  with a source citation. Time ceiling: 90 seconds per ADR-016 P-2.
- **Retrospective** (secondary): Persona 3 opens a scenario created by a colleague.
  She clicks the mode chip. `[data-testid="scenario-parameters"]` shows the full
  parameter set including entity, base year, n_steps, fiscal multiplier, and
  conditionality settings — or "(not recorded)" for values that predate parameter
  persistence.

**P-3 — Journey reference:**
- Closes Journey A Step 1 [Near-Term-Gap] — entity selection at scenario creation
  (Component 1 entity selector, already built in G1; G4 adds data quality preview)
- Closes Journey A Step 2 [Near-Term-Gap] — data quality disclosure at scenario creation
  (Component 1 data quality preview, G4)
- Introduces Journey B Step 0 (new, ADR-016) — input challenge response in Reactive
  state: Persona 2 opens Grounding strip, reads starting condition with source citation,
  produces response at the table within 90 seconds (Component 2, G4)
- Addresses Journey A Step 3 [Near-Term-Gap] — scenario parameter inspection
  post-creation (Component 4, G4)

**P-4 — Time/interaction ceiling:**
- Data quality preview: appears within 2 seconds of entity + year selection (no
  additional interaction). Source: ADR-016 §UX-3 criterion 1.
- Grounding strip: visible within 3 seconds of clicking "Grounding ▼" (1 interaction
  from Zone 0). Source: ADR-016 §UX-3 criterion 3.
- Reactive entry state input challenge response: Persona 2 reads a source-cited
  indicator within 90 seconds of receiving the challenge. Observable: from Zone 0 with
  scenario loaded, click "Grounding ▼," read indicator. Source: ADR-016 P-2 and P-4.
- Parameter display: visible within 1 interaction from Zone 0 (clicking mode chip).
  Source: ADR-016 §Component 4.

**P-6 — Negotiating leverage delivered:**
After G4, Persona 2 can make the following specific argument:

> "The model uses Jordan's reserve coverage as reported by the Central Bank of Jordan —
> 7.1 months as of Q4 2023. That figure is Tier 2 confidence — observed data, not
> synthetic. If the creditor side uses a different figure, they need to produce their
> source. Here is ours."

Previously: this argument was unavailable from the screen. G3 placed the data at the
API boundary. G4 puts it on screen in the Grounding strip. Neither was sufficient
without the other.

**P-7 — North star capability delivered:**
G4 delivers full P-7 for ADR-016 (closing G3's partial delivery at the API layer):
the Zambian finance ministry analyst can select her country in the creation form and
receive an immediate data quality signal, and in the session she can open the Grounding
strip and respond to an input challenge with a source citation from the tool itself —
within 90 seconds, without external documentation — closing the input-side asymmetry
that WorldSim's trust architecture (ADR-015 + ADR-016) is designed to close.

## 3. Observable Application State

### 3.1 Primary observable state

In the scenario creation form (opened via "Scenarios ▼"), with "JOR" selected in the
entity selector and "2024" entered as start year: the element
`[data-testid="data-quality-preview"]` is visible and contains:
- The text "JOR" (confirming the entity context)
- At least one confidence tier label matching the format `T[1-5]` (e.g., "T2")

The preview appears without any additional interaction (not gated behind a separate
button click) within 2 seconds of entity + year selection.

### 3.2 Secondary observable states

**Secondary state A — Grounding strip**

With the JOR scenario (`68b31277` or any JOR scenario with status `"completed"`) loaded
in Zone 1 (active scenario), clicking the "Grounding ▼" button causes
`[data-testid="grounding-strip"]` to become visible. Within 3 seconds of the click, the
element contains at least one indicator row where a source citation is non-empty. The
row format is: `[display_name]: [value] [unit]   [Source · Vintage · T{N}]`. The
"Grounding ▼" button must be visible in Zone 0 or Zone 2 without navigating away from
the primary viewport.

**Secondary state B — Parameter persistence display**

With any scenario loaded, clicking the mode indicator chip (element with
`data-testid="mode-indicator"` or its wrapper) causes `[data-testid="scenario-parameters"]`
to become visible. The element contains all four of: fiscal multiplier value, base year
value, entity code, and number of steps (n_steps). Values absent from the scenario's
stored configuration display the text "(not recorded)" in that row — the row is present
even for absent values.

**Secondary state C — Fidelity panel static header**

With the Fidelity panel open, the element `[data-testid="fidelity-contextualisation"]`
is visible at the top of the panel content (above the existing case cards). The element
contains text stating that this panel validates model relationships, not input data for
the active scenario. The exact required text is specified in §4b.

**Secondary state D — Choropleth reference header**

The choropleth panel (containing the world map) has a visible header element containing
the text "Reference data — not scenario outputs" verbatim (per ADR-016 §EL Decision 5).
This header is visible without any interaction — it is always-present, not
interaction-gated. It must be present whether a scenario is loaded or not.

**Secondary state E — Synthetic flag**

In the scenario creation form, with "ZMB" selected as entity and "2024" entered as start
year, the element `[data-testid="data-quality-preview"]` contains the word "synthetic"
verbatim. This is the explicit requirement of ADR-016 §Component 1 and
`docs/DATA_STANDARDS.md §Confidence Tier System` for T4 entries.

### 3.3 Silent failure detection

ADR-016 §Silent Failure Mode specifies verbatim fallback text for each component. A
silent failure (empty display without explanation) is **worse** than the pre-G4 state —
it implies the grounding strip or preview loaded successfully with zero data, rather than
disclosing an API failure. Distinguishing observable characteristics:

**Component 1 silent failure (API unavailable):**
If `/data-quality` returns an error or is unreachable, the creation form shows the text
"Data quality preview unavailable" (required verbatim prefix per ADR-016 §Silent Failure
Mode). An empty space at the preview location is a silent failure — distinguishable from
the fallback only if the fallback text is present. QA test: simulate 500 or network error
on `/data-quality`; assert `[data-testid="data-quality-preview"]` contains
"Data quality preview unavailable."

**Component 2 silent failure (empty frameworks):**
If `/initial-state` returns `{"frameworks": {}}` or `{"frameworks": {"None": ...}}` with
no named frameworks, the grounding strip shows the text "Initial state data not available
for this scenario" (required verbatim prefix per ADR-016 §Silent Failure Mode). An empty
grounding strip body is a silent failure — the user cannot distinguish a successful load
with zero indicators from an API failure. QA test: load the `cd60b9ee` legacy scenario
(empty entity list, empty `/initial-state` frameworks); assert grounding strip shows
fallback text.

**"None" framework filter (forwarded G3 observation):**
The `/initial-state` endpoint returns a `"None"` key in the `frameworks` object for
legacy simulation attributes lacking a `measurement_framework` field (`pop_rank`,
`economy_tier`, `gdp_usd_millions`). These must not render as a framework section in the
Grounding strip. Observable distinguisher: the Grounding strip contains no section heading
that is blank, the string "None", or the string "null" when scenario `68b31277` (JOR) is
loaded. Only the five named frameworks (financial, human_development, ecological,
governance, political_economy) may render as section headings.

## 4. Acceptance Criteria

**AC-1 (data quality preview — renders for JOR 2024):**
In the scenario creation form, when the entity selector is set to "JOR" and the start
year field is set to 2024, then `[data-testid="data-quality-preview"]` is visible in
the form, contains the text "JOR", and contains at least one tier label matching the
pattern `T[1-5]` (e.g., "T2" or "T3") — all within 2 seconds, without any additional
user interaction beyond the entity and year selection.

**AC-2 (data quality preview — synthetic flag for ZMB):**
In the scenario creation form, when the entity selector is set to "ZMB" and the start
year field is set to 2024, then `[data-testid="data-quality-preview"]` is visible and
contains the word "synthetic" verbatim (case-insensitive match is insufficient; the exact
lowercase string "synthetic" must be present per ADR-016 §Component 1 and DATA_STANDARDS.md).

**AC-3 (grounding strip — opens with indicator and source citation):**
With the JOR scenario (`68b31277` or a freshly created JOR completed scenario) selected
as the primary scenario in Zone 1, when the user clicks the "Grounding ▼" button, then
within 3 seconds `[data-testid="grounding-strip"]` is visible and contains at least one
text node that includes a non-empty source institution name (e.g., "CBJ", "IMF BOP",
"World Bank") — confirming a populated indicator row, not an empty or fallback state.

**AC-4 (parameter persistence — all four values visible in one click):**
With any completed scenario loaded as the primary scenario, when the user clicks the
mode indicator chip (or its clickable wrapper), then `[data-testid="scenario-parameters"]`
is visible and the element's text content includes all four of: a fiscal multiplier value
(e.g., "1.0" or "1.3"), a base year (a four-digit number between 1900 and 2100), an
entity code from {GRC, JOR, EGY, ZMB}, and the number of steps (a positive integer).
Rows where the stored value is absent display "(not recorded)" in that row — the row
itself must be present.

**AC-5 (fidelity contextualisation — static header present):**
With the Fidelity panel open (via "Fidelity ▼" or however the panel is accessed),
`[data-testid="fidelity-contextualisation"]` is visible and its text content contains
the phrases "model relationships" and "not input data" (both must be present, establishing
that the panel's scope clarification is correct per ADR-016 §EL Decision 2 and the
IC-4 M14 mitigation). The existing case cards and validation content must still be
present below this header — the header is additive, not a replacement.

**AC-6 (choropleth reference header — verbatim text):**
The choropleth panel (rendered by `ChoroplethMap.tsx` or its wrapper) contains a visible
element with the text "Reference data — not scenario outputs" verbatim (per ADR-016
§EL Decision 5). This text is present whether a scenario is loaded or not. The element
is visible without any interaction — it is not inside a collapsed panel or tooltip.

**AC-7 (grounding strip — "None" framework not rendered):**
With the JOR scenario (`68b31277`) loaded and the Grounding strip open, the strip
does not contain any section heading that is blank, the string "None", or the string
"null". The strip contains at least one named framework section (e.g., "Financial"
or "Human Development"). This confirms the `"None"` key filter is operating correctly.

**AC-8 (data quality preview — API unavailable fallback):**
When `GET /api/v1/entities/{entity_id}/data-quality` is unavailable (returns 500 or
network error — simulated via Playwright route mock), the creation form shows text
containing "Data quality preview unavailable" within the area where the preview would
appear — not an empty space, not an unhandled error. The form remains functional and
the user can still create a scenario.

**AC-9 (grounding strip — empty frameworks fallback):**
When `GET /api/v1/scenarios/{id}/initial-state` returns `{"frameworks": {}}` (simulated
via Playwright route mock or the legacy scenario `cd60b9ee`), the Grounding strip shows
text containing "Initial state data not available" — not an empty strip body, not an
unhandled error. The "Grounding ▼" button is still present and the panel still opens.

## 5. Kryptonite Constraint Check

> *Authority: CLAUDE.md §Agent Execution Lifecycle — Kryptonite Design Constraint (FD-3).*

**Does this implementation's primary observable state require specialist mediation for
Persona 2 to act on it in the Reactive entry state (90-second ceiling)?**

`[x]` No — the observable states are directly interpretable by Persona 2 without an
analyst translating them.

Justification per component:

- **Data quality preview (Component 1):** "T2 · IMF BOP · 2024-Q1" is a human-readable
  format. Tier 2 means observed data; IMF BOP is a named institution; 2024-Q1 is a
  calendar reference. A finance ministry economist does not require a specialist to
  understand these labels. "synthetic" (verbatim) is the disclosure — no translation
  needed.

- **Grounding strip (Component 2):** "Reserve coverage: 7.1 months · CBJ 2023-Q4 · T2"
  names the indicator in display terms ("Reserve coverage," not "reserve_coverage_months"),
  the value and unit, the source (Central Bank of Jordan), and the vintage (Q4 2023).
  The ministry team can cite "CBJ Q4 2023" at the negotiating table with the same
  evidentiary format as any Bloomberg or Article IV reference. No specialist mediation
  required.

- **Parameter persistence (Component 4):** "Entity: JOR · Base year: 2023 · Steps: 8 ·
  Fiscal multiplier: 1.30" names parameters in human-readable terms. No specialist
  translation required.

- **Static headers (IC-4, IC-6):** Plain-language text ("Reference data — not scenario
  outputs"; "validates model relationships, not input data"). No mediation required.

Kryptonite test: the ministry-side analyst with three economists can read the Grounding
strip entry and cite the source institution and vintage in response to a creditor challenge.
The creditor side uses the same citation format. No mediation asymmetry is introduced by G4.

### Initial BPO observation — REJECT on AC-3 (source citations absent)

BPO initial pass (before fix) detected that the Grounding strip rendered value and tier label only — `"Reserve coverage: 7.1 months · T2"` — with no source institution or vintage. `"CBJ"` and `"2023-Q4"` were absent. The P-6 negotiating leverage specified in §2 was not deliverable from the screen.

**Root cause:** `GroundingIndicator` in `types.ts` declared `source_institution`/`data_vintage` (the `/data-quality` endpoint field names) instead of `source`/`vintage` (the `/initial-state` endpoint field names per `api_contracts.yml`). `GroundingStrip.tsx` read `ind.source_institution` and `ind.data_vintage`, both `undefined` at runtime.

**Why Step 4 missed it:** The AC-3 E2E test used a generic regex fallback `/[·•]\s*\S/` that matched `· T2` (the tier separator), falsely confirming a "citation-like token" without detecting the missing institution name.

**Rejection artifact filed:** `docs/process/rejections/REJECT-001-2026-06-17-grounding-strip-missing-source-citations.md`
**Near-miss filed:** NM-045
**Data Architect loop-in:** EL-requested 2026-06-17. Data Architect confirmed `api_contracts.yml` is authoritative; backend correctly implements `source`/`vintage`; no contract update required.

**Note for §7 (Test Authorship Obligation) — field name contract:** For `GroundingIndicator`, the authoritative field names are `source` and `vintage` per `api_contracts.yml §GET /scenarios/{id}/initial-state`. These differ from `DataQualityFramework` which uses `source_institution`/`data_vintage` per `api_contracts.yml §GET /entities/{entity_id}/data-quality`. QA tests for AC-3 must assert string presence of institution name (e.g., "CBJ") by direct string match — not by character class regex.
